# 五子棋网页设计文档

## 目标

构建一个可在桌面浏览器中游玩的人机对战五子棋网页。第一版聚焦完整、清晰、可玩的单局体验，不做在线对战、用户账号、棋谱保存或复杂难度系统。

## 规则范围

- 采用自由规则五子棋。
- 棋盘为 15x15 交叉点网格。
- 人类玩家执黑，黑方先手。
- 电脑玩家执白，在人类玩家落子后自动回应。
- 横向、纵向或任一斜向形成五颗或以上连续同色棋子即胜利。
- 不采用禁手规则。
- 棋盘下满且无人胜利时为平局。
- 胜利或平局后进入棋局结束状态，棋盘不再接受落子。

## 对弈流程

1. 页面打开后直接展示空棋盘，状态为“轮到你落子”。
2. 人类玩家点击空交叉点落黑子。
3. 系统判断人类玩家是否胜利或是否平局。
4. 如果棋局未结束，状态切换为“电脑玩家思考中”。
5. 经过约 300-600ms 的短暂延迟后，电脑玩家落白子。
6. 系统判断电脑玩家是否胜利或是否平局。
7. 如果棋局未结束，状态回到“轮到你落子”。
8. 点击“重新开始”会清空棋盘并开始新棋局。

## 电脑玩家行为

第一版使用启发式电脑玩家，不追求专业棋力。

电脑玩家的落子优先级：

1. 如果电脑玩家存在一步胜利落点，选择该落点。
2. 如果人类玩家存在下一步直接胜利落点，优先封堵。
3. 否则根据简单评分选择较有利的位置。

评分应让电脑玩家表现得像在下棋，例如偏好靠近已有棋子的落点、延长自己的连续棋子、阻断人类玩家的连续棋子。第一版不要求多步搜索、难度选择或开局库。

## 界面设计

- 视觉风格改为赛博朋克风格，整体呈现霓虹、暗色、科技感和高对比界面。
- 页面背景使用深色城市夜景、网格光效或抽象电路纹理，避免纯色背景显得单薄。
- 主色建议使用深黑、蓝紫、青色霓虹和品红霓虹，适量使用黄色或红色作为警示强调色。
- 棋盘不再使用传统木色，改为半透明暗色面板、发光网格线和细微扫描线质感。
- 交叉点需要清晰可点，悬停时显示霓虹描边或光晕反馈。
- 黑棋建议设计为暗色金属质感并带冷色边缘光，白棋建议设计为亮银或浅色能量核心并带柔和外发光。
- 当前回合、胜负结果和“重新开始”按钮应具备统一的霓虹边框、微弱发光和简洁动效。
- 动效保持克制，只用于落子、电脑玩家思考中、胜利提示和按钮交互，不影响判断棋盘状态。
- 页面面向桌面浏览器，优先适配 1024px 及以上宽度。
- 棋盘建议约 640x640px，确保 15x15 交叉点具备足够点击空间。
- 状态区放在棋盘旁边或附近，保持简洁。
- 第一版不要求移动端体验。

## 状态区

状态区只显示当前状态或结果，不显示计时器、手数列表、棋谱或 AI 评分。

需要覆盖的文案：

- “轮到你落子”
- “电脑玩家思考中”
- “你赢了”
- “电脑玩家胜利”
- “平局”

## 操作范围

第一版只提供：

- 在空交叉点落子
- 重新开始

第一版不提供：

- 悔棋
- 先后手选择
- 难度选择
- 棋谱记录
- 计时器
- 在线对战
- 用户账号
- 排行榜

## 技术形态

第一版建议使用 Vite + React + TypeScript 构建前端：

- Vite 负责本地开发、构建和静态资源处理，启动快，适合轻量网页游戏。
- React 负责拆分棋盘、状态区、按钮和游戏状态展示组件，便于后续扩展界面细节。
- TypeScript 用于约束棋盘坐标、棋子颜色、棋局状态和电脑玩家返回值，降低规则逻辑出错概率。
- 样式建议使用普通 CSS Modules 或单独 CSS 文件，不需要引入大型 UI 组件库。
- 赛博朋克视觉主要通过 CSS 变量、阴影、渐变、纹理背景和关键帧动效实现。

不引入后端或数据库。棋局状态、胜负判断和电脑玩家逻辑都在浏览器内运行，构建产物应可作为静态网页部署。

## 验收标准

- 页面加载后能直接开始一局人机对战。
- 人类玩家只能在空交叉点落黑子。
- 人类玩家落子后，电脑玩家在短暂延迟后自动落白子。
- 任意方向五颗或以上连续同色棋子会立即判定胜利。
- 棋盘下满且无人胜利时判定平局。
- 棋局结束后棋盘不能继续落子。
- “重新开始”能清空棋盘并恢复到新棋局。
- 状态区能准确显示当前回合、电脑思考、胜利和平局。
- 1024px 及以上桌面宽度下棋盘和状态区可用且不重叠。
